# 公共交通空白地域の全国250mメッシュ分析

> 「実家に帰ると、車がないとコンビニにも行けない」
> 「親が運転をやめたら、買い物はどうする？」
> 「地元の友人はほぼ全員、車を持っていることが前提で生活している」

そんな地方の現実を、全国250mメッシュで可視化したのがこのプロジェクトです。

国土交通省の定義に基づき、鉄道駅・バス停への徒歩アクセスが困難な地域（公共交通空白地域）を全国規模で定量化します。

## 分析結果（令和2年国勢調査）

> **駅スナップ方式**: S12ホーム線形の両端点を駅ポイントとして使用（北口・南口等の複数出口に対応）。
> 各端点を最近傍道路ノード1点にスナップするため、カバー半径の過大評価なし。

| カテゴリ | 人口 | 割合 |
|---|---|---|
| 公共交通便利地域（鉄道駅 walk 1km以内） | **5,181万人** | 41.1% |
| 公共交通不便地域（バス停 walk 500m以内） | **5,503万人** | 43.7% |
| **公共交通空白地域** | **1,910万人** | **15.2%** |

**日本人の約7人に1人（1,910万人）が公共交通空白地域に住んでいる。**

65歳以上では **611万人**（高齢者全体の17.3%）が空白地域に居住。

## 国交省公表数値との比較

| 比較項目 | 国交省 QGIS手順（2025年4月） | 本試算 |
|---|---|---|
| **公共交通空白地域 人口** | **約735万人（5.8%）** | **約1,910万人（15.2%）** |
| 距離計算方法 | 直線距離（バッファ） | **道路距離（Dijkstra）** |
| 駅データ | N02（2023年） | **S12（2024年・ホーム線形両端点）** |
| バス停データ | P11（2022年） | P11（2022年） |
| メッシュ解像度 | 250m | 250m |
| 人口データ | 令和2年国勢調査 | 令和2年国勢調査 |

### 差異の主因：道路距離 vs 直線距離

本試算の空白地域人口（1,910万人）が国交省の公表値（735万人）の**約2.6倍**になっている主因は距離計算方法の違いです。

```
直線距離（国交省）       道路距離（本試算）

  駅                         駅
  ●──────────900m──────────○    ●                         ○
  直線900m < 1,000m              │ ←道路が迂回             │
  → 「便利地域」                  └──────1,300m──────────┘
                                 ネットワーク1,300m > 1,000m
                                 → 「空白地域」
```

道路が川・山・住宅地を迂回する場合、ネットワーク距離は直線距離の **1.2〜1.5倍** になることが多い。直線で800〜900m圏内の地域でも、実際の歩行距離は1km超となり「空白地域」に分類される。これが空白人口の乖離（約1,175万人分）の主要因です。

> ※ 国交省の数値は「国土数値情報とQGISを活用した交通空白地の抽出と人口分析」（2025年4月）より。直線距離バッファによる簡易推計値。

## 分析定義

国土交通省 国土数値情報QGISマニュアル（2025年4月）準拠。

> 出典: [国土数値情報とQGISを活用した交通空白地の抽出と人口分析](https://nlftp.mlit.go.jp/ksj/manual/QGIS_manual_01.htm)（国土交通省 政策統括官付 地理空間情報課、2025年4月）
>
> 参考: [「交通空白」解消に関する取組](https://www.mlit.go.jp/sogoseisaku/transport/sosei_transport_tk_000237.html)（国土交通省）

| カテゴリ | 定義 |
|---|---|
| **公共交通便利地域** | 最寄り鉄道駅まで徒歩1,000m以内 |
| **公共交通不便地域** | 鉄道駅1,000m超・最寄りバス停500m以内 |
| **公共交通空白地域** | 鉄道駅1,000m超 AND バス停500m超 |

距離は**道路ネットワーク上の徒歩距離**（walk 3.6 km/h）で計算する。

### 公式定義との相違点・優位点

| 項目 | 国交省公式手順 | 本分析 |
|---|---|---|
| 距離計算 | 直線距離 | **道路距離（Dijkstra）** |
| 駅データ | N02（全駅） | **S12（全駅・乗降客数情報付き）** |
| バス停データ | P11（2022年） | **P11（都道府県別最新）** |
| 駅スナップ | — | **半径500m以内の全道路ノード（複数出口対応）** |

道路距離により、川・山で隔てられた「直線は近いが実際には遠い」ケースを正確に捕捉できる。S12を使用することで駅名・運営者・乗降客数などの属性情報も保持できる。

### S12（駅別乗降客数）について

N02（鉄道路線）の代わりに S12 を使用。駅名・運営者・路線名・乗降客数の属性を保持できる。

- S12のジオメトリは線型（ホーム線形）→ **両端点**をポイントとして使用（北口・南口等の複数出口に対応）
- 乗降客数フィルタリングなし（全駅使用）
- 全国 **19,854 ポイント**（21,068 端点からジオメトリ重複除去後）

## アルゴリズム

```
[stations.parquet]          [busstops.parquet]
 S12ホーム線形の両端点               │
（北口・南口等の複数出口に対応）        │
        │                           │
        ▼                           ▼
  最近傍1道路ノードにスナップ    最近傍1道路ノードにスナップ
        │                           │
        └─────────┬──────────────────┘
                  ▼
        ┌─────────┴─────────┐
        ▼                   ▼
 Multi-source Dijkstra  Multi-source Dijkstra
   （駅 → 全ノード）       （バス停 → 全ノード）
        │                   │
        └─────────┬──────────┘
                  ▼
     アクセスリンク経由でメッシュ単位の時間に変換
     dist_to_station[mesh] = dist[road_node] + acc_time
     dist_to_bus[mesh]     = dist[road_node] + acc_time
                  │
                  ▼
     閾値判定（walk 3.6 km/h）
     バス停  > 8.3分（500m相当）  → far_bus
     鉄道駅  > 16.7分（1,000m相当）→ far_station
```

Multi-source Dijkstra はスーパーソースノード方式で実装。駅・バス停をそれぞれ一括投入し、1回の探索で全メッシュへの最短徒歩時間を算出する。

駅スナップにはS12ホーム線形の両端点を使用（デフォルト）。`--station-snap-radius`（単位: m）を指定すると各端点周辺の全道路ノードを追加スナップする（通常は不要）。

## 既知の制約

- **道路ネットワーク**: 車道中心線ベースのため、歩道専用路・公園内通路等は対象外
- **バス運行本数**: 本数が極端に少ない停留所も「バス停あり」として扱う（GTFSデータ未使用）
- **S12 乗降客数**: 2024年時点。廃線後のデータ更新ラグが生じる可能性がある
- **一方通行**: 道路ネットワークは双方向として処理（一方通行未考慮）
- **駅スナップ半径**: デフォルト500mは緯度方向の近似値。経度方向は約449m相当
